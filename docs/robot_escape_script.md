# 具体剧本

## 一、剧本说明

角色标记：

- **旁白**：用于场景说明和转场。
- **R-07**：玩家控制的小型维修机器人。
- **Echo**：工厂广播系统中残留的辅助程序。
- **MOTHER**：工厂中央控制系统。
- **巡逻者**：仍在执行封锁命令的安保机器人。
- **管理员日志**：事故前管理员留下的记录。
- **系统音**：工厂设备、终端和关卡系统的提示音。

剧本节奏与当前功能恢复顺序保持一致：

- 第 1 关：基础行动模块恢复，确认程序顺序。
- 第 2 关：计时与等待模块恢复，R-07 开始观察环境节奏。
- 第 3 关：判断与传感模块恢复，包括布尔、条件、循环、坐标与“前方能否通行”。
- 第 4 关：计算与记录模块恢复，包括运算和变量。
- 第 5 关：路径记录与复用模块恢复，包括列表、自定义积木、哈希与取余。
- 第 6 关：数据输出与整理模块恢复，包括输出和排序。
- 第 7 关：编码解析模块恢复，对应字符串处理。
- 第 8、9 关：不再恢复新的基础功能，转为综合运用。

**关卡开始对话：**

**R-07：**

**Echo：**

**R-07：**

**R-07：**

**任务提示：**

**代码引导：**

本关重点是处理列表数据。遍历 `时间` 和 `位置` 两个列表，先跳过无效请求，再按 `时间` 从小到大选出目标放入 `有效结果` 列表。若时间相同，位置较小的请求排在前面。

例如 n=4，时间列表为 `[7, 3, -1, 12]`，位置列表为 `[41, 284, 10, -1]`。第 1 项 (7, 41) 和第 2 项 (3, 284) 有效；第 3 项时间 = -1 无效，跳过；第 4 项位置 = -1 无效，跳过。按时间从小到大排序：时间 3 的请求对应位置 284，时间 7 的请求对应位置 41。最终有效结果为 `[284, 41]`。

## 九、第 7 关：倒放的最后日志

**场景：**

数据中继室下方有一台管理员终端。终端保存着最后日志，但日志被加密成一串数字。加密方式为：首先倒序，之后每个数字加 3，数字 10 变成 0，最后整体循环向右移动 2 位。
保证密码明文开头不是0

例如：

`923456 -> 654329 -> 987652 -> 529876`

**关卡开始对话：**

**R-07：**

检测到管理员日志。内容为数字序列。状态：加密。

**Echo：**

他们知道这段话不能被轻易删掉，所以把它藏进了错误的顺序里。

**R-07：**

加密步骤：倒序，数字加三，循环右移两位。

**Echo：**

要恢复它，就把步骤倒着走。

**R-07：**

解密步骤：循环左移两位，数字减三，最后倒序。

**任务提示：**

编码解析模块恢复，管理员日志补充字符串内容；但实际测试仍以数字变量为核心。根据加密规则反向解密数字序列，把恢复出的明文写入变量 `明文`。

**代码引导：**

先把变量 `密码` 拆成一位一位的数字来处理。解密顺序必须和加密顺序相反：先循环左移两位，再每位减三并处理 0 到 9 的回绕，最后倒序组合回变量 `明文`。建议用列表保存每一位，逐步修改列表，比直接操作整个数字更稳。

**玩家成功后：**

**管理员日志：**

若有维修单元醒来，请带出日志。

**管理员日志：**

但不要直接开门。先确认冷却系统是否稳定。

**管理员日志：**

母机只会执行封锁命令。它无法判断外界是否安全。

**R-07：**

目标更新：离开工厂，带出日志，确认冷却稳定。

**Echo：**

**R-07：**

Echo，你是管理员留下的程序？

短暂静默。

**Echo：**

我是他没来得及说完的话。

**失败反馈：**

**系统音：**

**R-07：**

**场景：**

- 广播天线：需要 R-07 站在天线控制点旁的踏板上等待，`天线稳定度` 每次等待增加 1，达到 5 后日志可发送。
- 冷却系统：初始 `冷却值 = 30`，时间推进时会持续下降；在冷却点旁等待时，`冷却值` 上升 5；任何时候 `冷却值 < 0` 都会失败。
- 时间窗口：只有 60 帧。超过 60 帧后会立刻结算；只完成阀门或天线会进入对应不完美结局，两个都没完成则失败，关卡进度仍停留在第 9 关。
